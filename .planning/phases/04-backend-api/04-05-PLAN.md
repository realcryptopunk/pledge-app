<task type="auto">
  <name>Task 2: Add Supabase Realtime subscriptions for habits</name>
  <files>Core/AppState.swift</files>
  <action>
Add Supabase Realtime subscriptions so habits update live when changed from another device or from Supabase dashboard.

In AppState, after the SupabaseService is initialized (when authenticated client arrives), set up a Realtime channel:

```swift
private var habitsChannel: RealtimeChannelV2?

func setupRealtimeSubscriptions() async {
    guard let client = supabaseClient else { return }

    // Subscribe to changes on the habits table for the current user
    let channel = client.realtimeV2.channel("habits-changes")

    let changes = channel.postgresChange(
        AnyAction.self,
        schema: "public",
        table: "habits"
    )

    await channel.subscribe()

    // Listen for changes
    Task {
        for await change in changes {
            await handleHabitChange(change)
        }
    }

    habitsChannel = channel
}
```

Handle changes:
```swift
private func handleHabitChange(_ change: AnyAction) async {
    // Reload all habits from Supabase on any change
    // This is simpler than handling INSERT/UPDATE/DELETE individually
    // and ensures consistency
    guard let service = supabaseService else { return }
    do {
        let updatedHabits = try await service.fetchHabits()
        await MainActor.run {
            self.habits = updatedHabits
            self.generateTodayHabits()
        }
    } catch {
        print("Realtime habit refresh failed: \(error)")
    }
}
```

Call `setupRealtimeSubscriptions()` after SupabaseService is created (in the Combine sink from 04-04 Task 2).

On sign out, unsubscribe:
```swift
func teardownRealtimeSubscriptions() async {
    if let channel = habitsChannel {
        await channel.unsubscribe()
        habitsChannel = nil
    }
}
```

Call `teardownRealtimeSubscriptions()` in `signOut()`.

NOTE: The Supabase Swift SDK v2 Realtime API may have slightly different syntax — check the actual SDK version. The key pattern is: create channel → subscribe to postgres changes → handle changes in async for loop. Adapt to whatever the actual SDK API is.

Do NOT set up Realtime for user_profiles or habit_logs — habits are the primary use case. Portfolio balance updates can be added later if needed.
