**Edge function** (`supabase/functions/leaderboard/index.ts`):

Create a leaderboard endpoint that returns ranked users. Accepts query parameters for different ranking types.

GET `/functions/v1/leaderboard?type=streaks&limit=50`
GET `/functions/v1/leaderboard?type=consistency&limit=50&period=30`
GET `/functions/v1/leaderboard?type=staked&limit=50`

Types:
- `streaks` — Ranked by highest active streak across all habits. Query:
  ```sql
  SELECT up.id, up.username, up.display_name, up.avatar_url,
         MAX(h.current_streak) as best_streak,
         COUNT(h.id) as active_habits
  FROM user_profiles up
  JOIN habits h ON h.user_id = up.id AND h.is_active = true
  WHERE up.username IS NOT NULL
  GROUP BY up.id
  ORDER BY best_streak DESC
  LIMIT $limit
  ```

- `consistency` — Ranked by verification rate over last N days. Query:
  ```sql
  SELECT up.id, up.username, up.display_name, up.avatar_url,
         COUNT(CASE WHEN hl.status = 'verified' THEN 1 END)::float /
         NULLIF(COUNT(hl.id), 0) as consistency_rate,
         COUNT(CASE WHEN hl.status = 'verified' THEN 1 END) as verified_count
  FROM user_profiles up
  JOIN habit_logs hl ON hl.user_id = up.id AND hl.log_date >= CURRENT_DATE - ($period || ' days')::interval
  WHERE up.username IS NOT NULL
  GROUP BY up.id
  ORDER BY consistency_rate DESC
  LIMIT $limit
  ```

- `staked` — Ranked by total amount staked (invested from failures). Query:
  ```sql
  SELECT up.id, up.username, up.display_name, up.avatar_url,
         COALESCE(SUM(hl.invested_amount), 0) as total_invested,
         up.investment_pool_balance
  FROM user_profiles up
  LEFT JOIN habit_logs hl ON hl.user_id = up.id AND hl.status = 'failed'
  WHERE up.username IS NOT NULL
  GROUP BY up.id
  ORDER BY total_invested DESC
  LIMIT $limit
  ```

Response format:
```json
{
  "type": "streaks",
  "entries": [
    { "rank": 1, "user_id": "uuid", "username": "jake", "display_name": "Jake M.", "avatar_url": null, "value": 45, "label": "45 day streak", "active_habits": 3 },
    ...
  ]
}
```

Use the Supabase service_role key to query across users (bypasses RLS).
Deploy: `supabase functions deploy leaderboard`
