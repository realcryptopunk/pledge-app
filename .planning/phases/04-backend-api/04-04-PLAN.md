**Codable DTOs:** Create lightweight Codable structs that map to Supabase table columns. Use `snake_case` CodingKeys since Postgres columns are snake_case. These can be nested structs inside SupabaseService or in Models.swift:

- `HabitDTO` — maps Habit ↔ `habits` table. Include all Habit properties + `user_id`. The `id` should be the same UUID used locally.
- `HabitLogDTO` — maps HabitLog ↔ `habit_logs` table. Include `user_id`.
- `UserProfileDTO` — maps to `user_profiles` for reading/updating profile data (vault_balance, investment_pool_balance, risk_profile).

Add snake_case CodingKeys to each DTO. Example:
```swift
struct HabitDTO: Codable {
    let id: UUID
    let userId: UUID
    var name: String
    // ... all fields
    enum CodingKeys: String, CodingKey {
        case id, name, icon, type
        case userId = "user_id"
        case stakeAmount = "stake_amount"
        // etc.
    }
}
```

Use the Supabase Swift SDK query builder:
```swift
func fetchHabits() async throws -> [Habit] {
    let dtos: [HabitDTO] = try await client.from("habits")
        .select()
        .order("created_at")
        .execute()
        .value
    return dtos.map { $0.toHabit() }
}

func createHabit(_ habit: Habit) async throws {
    let dto = HabitDTO(from: habit, userId: userId)
    try await client.from("habits")
        .insert(dto)
        .execute()
}
```

Add conversion methods between DTOs and domain models (Habit ↔ HabitDTO). Keep the domain Habit model unchanged — DTOs handle the mapping.

**Initialize SupabaseService when authenticated client arrives:**
Subscribe to the supabaseClient publisher (set up in 04-03). When a non-nil client arrives, create SupabaseService and trigger initial data load:

```swift
$supabaseClient
    .compactMap { $0 }
    .sink { [weak self] client in
        guard let self else { return }
        self.supabaseService = SupabaseService(client: client)
        Task { await self.loadFromSupabase() }
    }
    .store(in: &cancellables)
```

**Replace saveHabits()/loadHabits():**

Keep the existing UserDefaults methods as `saveHabitsLocally()` / `loadHabitsLocally()` for guest mode.

Create new methods:
```swift
func loadFromSupabase() async {
    guard let service = supabaseService else {
        loadHabitsLocally()
        return
    }
    do {
        habits = try await service.fetchHabits()
        let profile = try await service.fetchUserProfile()
        vaultBalance = profile.vaultBalance
        investmentPoolBalance = profile.investmentPoolBalance
        // risk profile is @AppStorage, update if server differs
        if let serverProfile = RiskProfile(rawValue: profile.riskProfile) {
            riskProfile = serverProfile
        }
        generateTodayHabits()
    } catch {
        print("Supabase load failed, falling back to local: \(error)")
        loadHabitsLocally()
    }
}
```

**Update CRUD methods** (addHabit, updateHabit, deleteHabit, togglePauseHabit):
After modifying the local `habits` array, also persist to Supabase if service is available:
```swift
func addHabit(_ habit: Habit) {
    habits.append(habit)
    generateTodayHabits()
    if let service = supabaseService {
        Task { try? await service.createHabit(habit) }
    } else {
        saveHabitsLocally()
    }
}
```

Follow same pattern for updateHabit, deleteHabit, togglePauseHabit — local-first update, then sync to Supabase async. This gives instant UI response.

// After setting status
if let service = supabaseService {
