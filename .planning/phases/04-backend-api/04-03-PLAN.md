  <action>
Create `supabase/functions/auth-bridge/index.ts` that:

1. Receives POST with `{ privy_token, wallet_address, phone }` from iOS
2. Verifies the Privy token:
   - Call Privy's token verification endpoint OR verify the JWT signature using Privy's JWKS endpoint (`https://auth.privy.io/.well-known/jwks.json`)
   - Extract `sub` claim (Privy user ID) from the verified token
   - Alternatively, use Privy's REST API: `GET https://auth.privy.io/api/v1/users/{privy_user_id}` with app secret to verify user exists
3. Upsert into `user_profiles` table using service_role client (bypasses RLS):
   ```sql
   INSERT INTO user_profiles (id, privy_user_id, phone, wallet_address)
   VALUES (gen_random_uuid(), $privy_user_id, $phone, $wallet_address)
   ON CONFLICT (privy_user_id) DO UPDATE SET
     phone = EXCLUDED.phone,
     wallet_address = EXCLUDED.wallet_address,
     updated_at = now()
   RETURNING id;
   ```
4. Mint a custom Supabase JWT:
   - Sign with `SUPABASE_JWT_SECRET` (available in edge function env)
   - Claims: `{ sub: user_profile_id, role: "authenticated", aud: "authenticated", exp: now + 1hr, iat: now }`
   - The `sub` claim MUST match the `id` column in `user_profiles` so `auth.uid()` in RLS resolves correctly
5. Return `{ access_token: <JWT>, user_id: <UUID>, expires_at: <timestamp> }`

Use `jose` library for JWT operations (import from npm: `jose`).

Environment variables (set via `supabase secrets set`):
- `SUPABASE_JWT_SECRET` — project's JWT secret (from Supabase dashboard → Settings → API → JWT Secret)
- `PRIVY_APP_ID` — for token verification
- `PRIVY_APP_SECRET` — for Privy API calls
- `SUPABASE_SERVICE_ROLE_KEY` — for upserting user_profiles

CORS: Allow `*` for development.

Error handling:
- 401 if Privy token is invalid/expired
- 500 if JWT minting fails
- 409 if user profile creation conflicts unexpectedly

Deploy: `supabase functions deploy auth-bridge`
Set secrets for Privy app secret and JWT secret.

**SupabaseConfig.swift** — Add authenticated client support:

Add a method or property that creates/returns a SupabaseClient configured with a custom access token provider. The Supabase Swift SDK v2 supports this via the `accessToken` parameter in `SupabaseClientOptions.AuthOptions`:

```swift
enum SupabaseConfig {
    static let url = URL(string: EnvConfig.supabaseURL)!
    static let anonKey = EnvConfig.supabaseAnonKey

    // Unauthenticated client (for public operations)
    static let client = SupabaseClient(supabaseURL: url, supabaseKey: anonKey)

    // Authenticated client with custom JWT
    @MainActor
    static func authenticatedClient(accessToken: @escaping () async -> String) -> SupabaseClient {
        SupabaseClient(
            supabaseURL: url,
            supabaseKey: anonKey,
            options: .init(
                auth: .init(
                    accessToken: accessToken
                )
            )
        )
    }
}
```

**PrivyManager.swift** — Add Supabase auth bridge call:

After successful `verifyOTP()` (line ~115, after wallet creation), call the auth-bridge edge function:

```swift
@Published var supabaseAccessToken: String?
@Published var supabaseUserId: String?

private func authenticateWithSupabase() async throws {
    guard let user = authenticatedUser else { return }

    let url = URL(string: "\(EnvConfig.supabaseURL)/functions/v1/auth-bridge")!
    var request = URLRequest(url: url)
    request.httpMethod = "POST"
    request.setValue("Bearer \(EnvConfig.supabaseAnonKey)", forHTTPHeaderField: "Authorization")
    request.setValue("application/json", forHTTPHeaderField: "Content-Type")

    // Get Privy access token from the authenticated user
    let privyToken = try await user.getAccessToken()

    let body: [String: Any] = [
        "privy_token": privyToken,
        "wallet_address": walletAddress ?? "",
        "phone": userPhone ?? ""
    ]
    request.httpBody = try JSONSerialization.data(withJSONObject: body)

    let (data, _) = try await URLSession.shared.data(for: request)
    let json = try JSONSerialization.jsonObject(with: data) as? [String: Any]

    supabaseAccessToken = json?["access_token"] as? String
    supabaseUserId = json?["user_id"] as? String
}
```

Call `authenticateWithSupabase()` inside `verifyOTP()` after `createWalletIfNeeded()` and before setting `isAuthenticated = true`. Also call it in `checkAuthState()` if user is already authenticated (app relaunch).

Handle the case where the user's Supabase token expires — add a `refreshSupabaseToken()` method that re-calls the bridge. Check token expiry before Supabase operations.
