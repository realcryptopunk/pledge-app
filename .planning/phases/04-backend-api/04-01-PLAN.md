**Tables:**

1. `user_profiles` — Primary user table keyed by UUID, linked to Privy via `privy_user_id`
   - `id UUID PRIMARY KEY DEFAULT gen_random_uuid()`

   - `risk_profile TEXT NOT NULL DEFAULT 'moderate'` — conservative/moderate/aggressive
   - `vault_balance DECIMAL(18,2) DEFAULT 0`
   - `investment_pool_balance DECIMAL(18,2) DEFAULT 0`
   - `created_at TIMESTAMPTZ DEFAULT now()`
   - `updated_at TIMESTAMPTZ DEFAULT now()`

2. `habits` — User habits synced from iOS
   - `id UUID PRIMARY KEY DEFAULT gen_random_uuid()`
   - `user_id UUID NOT NULL REFERENCES user_profiles(id) ON DELETE CASCADE`
   - `name TEXT NOT NULL`
   - `icon TEXT NOT NULL`
   - `type TEXT NOT NULL` — matches HabitType raw values
   - `stake_amount DECIMAL(18,2) DEFAULT 10`
   - `schedule INT[] DEFAULT ARRAY[1,2,3,4,5,6,7]`
   - `target_value DECIMAL DEFAULT 0`
   - `verification_type TEXT DEFAULT 'auto'`
   - `is_active BOOLEAN DEFAULT true`
   - `is_paused BOOLEAN DEFAULT false`
   - `current_streak INT DEFAULT 0`
   - `best_streak INT DEFAULT 0`
   - `success_rate DECIMAL DEFAULT 0`
   - `location_latitude DECIMAL`
   - `location_longitude DECIMAL`
   - `location_radius DECIMAL`
   - `location_name TEXT`
   - `created_at TIMESTAMPTZ DEFAULT now()`
   - `updated_at TIMESTAMPTZ DEFAULT now()`

3. `habit_logs` — Every daily verification result
   - `id UUID PRIMARY KEY DEFAULT gen_random_uuid()`
   - `habit_id UUID NOT NULL REFERENCES habits(id) ON DELETE CASCADE`
   - `user_id UUID NOT NULL REFERENCES user_profiles(id) ON DELETE CASCADE`
   - `log_date DATE NOT NULL`
   - `status TEXT NOT NULL DEFAULT 'pending'` — verified/failed/skipped/pending
   - `verified_at TIMESTAMPTZ`

   - `verification_data JSONB` — flexible metadata (photo hash, health data, location)
   - `created_at TIMESTAMPTZ DEFAULT now()`
   - UNIQUE constraint on `(habit_id, log_date)` to prevent duplicate daily logs

4. `deposits` — Coinbase Onramp deposits
   - `id UUID PRIMARY KEY DEFAULT gen_random_uuid()`
   - `user_id UUID NOT NULL REFERENCES user_profiles(id)`
   - `session_id TEXT` — Coinbase session token ID
   - `amount DECIMAL(18,2) NOT NULL`
   - `status TEXT DEFAULT 'pending'` — pending/confirmed/failed
   - `wallet_address TEXT NOT NULL`
   - `coinbase_response JSONB`
   - `created_at TIMESTAMPTZ DEFAULT now()`
   - `confirmed_at TIMESTAMPTZ`

5. `transactions` — Activity feed / audit trail
   - `id UUID PRIMARY KEY DEFAULT gen_random_uuid()`
   - `user_id UUID NOT NULL REFERENCES user_profiles(id)`
   - `type TEXT NOT NULL` — deposit/stake_invested/penalty/yield/withdrawal
   - `amount DECIMAL(18,2) NOT NULL`
   - `description TEXT`
   - `related_habit_id UUID REFERENCES habits(id)`
   - `created_at TIMESTAMPTZ DEFAULT now()`

6. `friendships` — Social connections
   - `id UUID PRIMARY KEY DEFAULT gen_random_uuid()`
   - `user_id UUID NOT NULL REFERENCES user_profiles(id)`
   - `friend_id UUID NOT NULL REFERENCES user_profiles(id)`
   - `status TEXT DEFAULT 'pending'` — pending/accepted/blocked
   - `created_at TIMESTAMPTZ DEFAULT now()`
   - UNIQUE constraint on `(user_id, friend_id)`
   - CHECK constraint `user_id != friend_id`

**Indexes:**
- `habits(user_id)` — fast lookup by user
- `habit_logs(user_id, log_date)` — daily verification queries
- `habit_logs(habit_id, log_date)` — per-habit history
- `transactions(user_id, created_at DESC)` — activity feed
- `deposits(user_id, status)` — pending deposit lookup
- `user_profiles(username)` — username search
- `friendships(user_id, status)` and `friendships(friend_id, status)` — friend lookup

**RLS Policies (enable RLS on ALL tables):**

For `user_profiles`:
- SELECT: `auth.uid() = id` OR any authenticated user can read username/display_name/avatar_url/current streaks (for leaderboard)
- UPDATE: `auth.uid() = id` (own profile only)
- INSERT: `auth.uid() = id` (own profile only)

For `habits`:
- ALL operations: `auth.uid() = user_id`

For `habit_logs`:
- SELECT, INSERT: `auth.uid() = user_id`
- UPDATE: `auth.uid() = user_id`

For `deposits`:
- SELECT, INSERT: `auth.uid() = user_id`

For `transactions`:
- SELECT: `auth.uid() = user_id`
- INSERT: `auth.uid() = user_id`

For `friendships`:
- SELECT: `auth.uid() = user_id OR auth.uid() = friend_id`
- INSERT: `auth.uid() = user_id`
- UPDATE: `auth.uid() = friend_id` (only friend can accept)
- DELETE: `auth.uid() = user_id OR auth.uid() = friend_id`

**Leaderboard view** (materialized or regular):
```sql
CREATE VIEW leaderboard AS
SELECT
  up.id, up.username, up.display_name, up.avatar_url,
  COALESCE(MAX(h.current_streak), 0) as best_active_streak,
  COUNT(DISTINCT h.id) as active_habits,
  COALESCE(SUM(CASE WHEN hl.status = 'verified' THEN 1 ELSE 0 END), 0) as total_verified,
  COALESCE(SUM(hl.invested_amount), 0) as total_invested
FROM user_profiles up
LEFT JOIN habits h ON h.user_id = up.id AND h.is_active = true
LEFT JOIN habit_logs hl ON hl.user_id = up.id AND hl.log_date >= CURRENT_DATE - INTERVAL '30 days'
WHERE up.username IS NOT NULL
GROUP BY up.id;
```
