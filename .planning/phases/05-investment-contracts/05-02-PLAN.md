    Create `contracts/test/PledgeVault.t.sol` using Foundry's Test base:

    Setup:
    - Deploy MockUSDC, MockPT, MockPendleRouter, PledgeVault
    - Pre-mint MockPT supply to MockPendleRouter (so it can "return" PT from swaps)
    - Set relayer address (use makeAddr("relayer"))
    - Set active market (use makeAddr("market"))
    - Create test user addresses

    Test categories:

    **Access Control (4 tests):**
    - test_setRelayer_onlyOwner — owner can set, non-owner reverts
    - test_setActiveMarket_onlyOwner — owner can set, non-owner reverts
    - test_investForUser_onlyRelayer — relayer can call, non-relayer reverts
    - test_investForUser_revertsWhenNoMarket — reverts if activeMarket not set

    **Investment Flow (4 tests):**
    - test_investForUser_pullsUSDC — USDC transferred from user to vault, then to router
    - test_investForUser_creditsPtBalance — userPtBalance incremented by PT received
    - test_investForUser_emitsEvent — Invested event emitted with correct args
    - test_investForUser_multipleInvestments — two investments accumulate PT balance

    **Emergency Withdraw (3 tests):**
    - test_emergencyWithdraw_sendsPT — PT transferred to user, balance zeroed
    - test_emergencyWithdraw_emitsEvent — EmergencyWithdraw event emitted
    - test_emergencyWithdraw_revertsWhenZeroBalance — reverts if no PT balance

    **Edge Cases (3 tests):**
    - test_investForUser_revertsZeroAmount — reverts on 0 USDC
    - test_investForUser_revertsZeroAddress — reverts on address(0) user
    - test_investForUser_revertsInsufficientApproval — reverts if user hasn't approved USDC

    **Slippage (1 test):**
    - test_investForUser_respectsMinPtOut — reverts if PT output below minPtOut (configure mock to return less)

    Test patterns:
    - Use vm.prank(address) for caller impersonation
    - Use vm.expectRevert() for revert assertions
    - Use vm.expectEmit() for event assertions
    - Use deal() or mock mint for token balances
    - Each test: setup → action → assertion (Arrange-Act-Assert)

    Avoid: Do NOT test OpenZeppelin internals (ERC20, Ownable). Do NOT write fuzz tests yet (defer to pre-audit phase). Do NOT test Pendle Router behavior (that's what mocks isolate).
