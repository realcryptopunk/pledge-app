<task type="auto">
  <name>Task 2: Implement PledgeVault.sol and IPendleRouter interface</name>
  <files>contracts/src/PledgeVault.sol, contracts/src/interfaces/IPendleRouter.sol</files>
  <action>
    1. Create `contracts/src/interfaces/IPendleRouter.sol`:
       - Define minimal interface with `swapExactTokenForPt` function signature
       - Include struct definitions: ApproxParams, TokenInput, SwapData, LimitOrderData
       - Match Pendle Router V4 interface exactly (see DISCOVERY.md)

    2. Create `contracts/src/PledgeVault.sol`:
       - Inherit: Ownable, ReentrancyGuard (from OpenZeppelin)
       - Immutables: IERC20 usdc, IPendleRouter pendleRouter
       - State: address relayer, address activeMarket, address activePtToken
       - Mapping: userPtBalance (address => uint256)
       - Events: Invested(user, usdcAmount, ptReceived), EmergencyWithdraw(user, ptAmount), MarketUpdated(market, pt), RelayerUpdated(relayer)

       Functions:
       a) constructor(address _usdc, address _pendleRouter) — set immutables, msg.sender as owner
       b) setRelayer(address _relayer) — onlyOwner, emit RelayerUpdated
       c) setActiveMarket(address _market, address _pt) — onlyOwner, emit MarketUpdated
       d) investForUser(address user, uint256 usdcAmount, uint256 minPtOut) — onlyRelayer, nonReentrant
          - require(user != address(0), usdcAmount > 0, activeMarket != address(0))
          - SafeERC20.safeTransferFrom(usdc, user, address(this), usdcAmount) — pulls from user
          - SafeERC20.safeApprove(usdc, address(pendleRouter), usdcAmount)
          - Build TokenInput struct (tokenIn=usdc, netTokenIn=usdcAmount, tokenMintSy=address(usdc), pendleSwap=address(0), swapData=empty)
          - Build default ApproxParams (guessMin=0, guessMax=type(uint256).max, guessOffset=0, maxIteration=256, eps=1e15)
          - Call pendleRouter.swapExactTokenForPt(address(this), activeMarket, minPtOut, approxParams, tokenInput, emptyLimitOrder)
          - userPtBalance[user] += netPtOut
          - Emit Invested(user, usdcAmount, netPtOut)
       e) emergencyWithdraw() — nonReentrant
          - uint256 balance = userPtBalance[msg.sender]
          - require(balance > 0)
          - userPtBalance[msg.sender] = 0
          - SafeERC20.safeTransfer(IERC20(activePtToken), msg.sender, balance)
          - Emit EmergencyWithdraw(msg.sender, balance)
       f) getUserPtBalance(address user) — view, returns uint256
       g) modifier onlyRelayer — require(msg.sender == relayer)

       Security:
       - Use SafeERC20 for all token operations (handles non-standard return values)
       - ReentrancyGuard on investForUser and emergencyWithdraw
       - Check-effects-interactions pattern: update state before external calls
       - Slippage protection via minPtOut parameter (never 0 in production)
       - No approve(type(uint256).max) — approve exact amount per swap

    Avoid: Do NOT implement ERC-4626 (overkill for per-user tracking). Do NOT add withdrawal timelock yet (defer to Phase 6). Do NOT hardcode addresses — pass via constructor.
  </action>
