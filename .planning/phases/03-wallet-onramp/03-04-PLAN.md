**Coinbase Onramp integration:**
- Hosted URL: `https://pay.coinbase.com/buy/select-asset`
- Required params: `appId` (from Coinbase Developer Platform), `addresses` (JSON: {"wallet_address":["base"]}), `assets` (["USDC"]), `presetFiatAmount`, `fiatCurrency` (USD)
- Optional params: `defaultNetwork` (base or arbitrum), `defaultAsset` (USDC), `handlingRequestedUrls` (true)
- The onramp opens in WKWebView — user completes purchase flow entirely within Coinbase's UI
- On completion, crypto is sent to the wallet address. No callback URL needed for MVP — we can poll balance later.
- For production: session tokens are required (generated server-side). For MVP/development, the URL-based approach works without tokens.

**Current deposit flow:**
- SetupDepositView: amount selector with number pad, quick pills ($50-$500), mock "Apple Pay" button
- On "Pay": just sets flowState.depositAmount and advances to next step
- No real payment processing

**Target flow:**
1. User selects amount in SetupDepositView (keep existing UI — it's great)
2. Instead of mock "Pay" button, show "Fund with Coinbase" button
3. Button presents CoinbaseOnrampView as a sheet
4. CoinbaseOnrampView loads Coinbase Onramp URL in WKWebView with wallet address + amount
5. User completes purchase in Coinbase flow
6. On dismiss, advance setup flow

**Design notes:**
- Keep the existing number pad and amount selector — it works well
- Replace the Apple Pay button with a Coinbase-branded button (blue background, Coinbase logo or text)
- Keep "Or pay with card" as fallback (can also open Coinbase)
- Security note text stays
</context>

- Construct Coinbase Onramp URL with query parameters:
  ```
  https://pay.coinbase.com/buy/select-asset
    ?appId=YOUR_COINBASE_APP_ID  (placeholder — TODO comment)
    ?addresses={"walletAddress":["base"]}  (URL-encoded JSON with the user's embedded wallet address)
    ?assets=["USDC"]
    ?presetFiatAmount={amount}

    ?defaultNetwork=base
    ?defaultAsset=USDC
  ```
- Add top bar with "Fund Your Wallet" title and X close button (calls onDismiss)
- Loading indicator while page loads (ProgressView overlay that hides when WKWebView finishes loading)
- WKNavigationDelegate to handle:
  - `decidePolicyFor`: Allow all Coinbase URLs. If URL matches a completion/success pattern, call onDismiss.
  - `didFinish`: Hide loading indicator
  - `didFail`: Show error state with retry button

**UIViewRepresentable implementation:**
- Create WKWebViewContainer: UIViewRepresentable
- makeUIView: create WKWebView with proper configuration (javaScriptEnabled, allowsInlineMediaPlayback)
- updateUIView: load the URL request
- Coordinator class as WKNavigationDelegate

**Important:** URL-encode the `addresses` JSON parameter properly. The addresses format Coinbase expects is: `{"0xABC123...":["base"]}` where the key is the wallet address.
