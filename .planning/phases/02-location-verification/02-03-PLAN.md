<task type="auto">
  <name>Task 1: Create LocationPickerView</name>
  <files>Features/Habits/LocationPickerView.swift</files>
  <action>
Create a location picker view using MapKit that lets users search for and select a location.

**Structure:**
```
struct LocationPickerView: View {
    @Binding var latitude: Double?
    @Binding var longitude: Double?
    @Binding var radius: Double?
    @Binding var locationName: String?
    @Environment(\.dismiss) private var dismiss
    @Environment(\.themeColors) var theme
}
```

**Layout — presented as a sheet:**
1. NavigationStack with title "Set Location"
2. Toolbar: Cancel (leading, ghost), Done (trailing, only enabled when location selected)
3. Content:
   - Search bar at top — `TextField` with magnifying glass icon, styled with `.ultraThinMaterial` background and rounded rect. On submit, use `MKLocalSearch` to find places.
   - Map view — `Map` (SwiftUI MapKit, iOS 17+) showing the selected region. Use `@State private var position: MapCameraPosition`. Show an `Annotation` at the selected coordinate with a themed pin.
   - Search results list — when search returns results, show them in a VStack over the map with `.cleanCard()` styling. Each result shows name + address. Tapping a result selects it, updates the map position, and dismisses the results list.
   - Radius control — below the map, a labeled stepper/slider: "Radius: [X]m" with presets [100, 150, 200, 300, 500]. Default 150m. Use capsule pill buttons like the stake amount presets in AddHabitView.
   - Selected location display — below radius, show the locationName and coordinates in a `.cleanCard()`.

**Search implementation:**
- Use `MKLocalSearch.Request()` with `naturalLanguageQuery` from the search text
- Set `region` to the visible map region for better results
- Store results as `[MKMapItem]`
- On result tap: set lat/lng from `placemark.coordinate`, set locationName from `placemark.name ?? placemark.title`

**Map interaction:**
- Use `MapReader` to handle tap gestures on the map for manual pin placement
- When user taps the map (not a search result), reverse geocode with `CLGeocoder` to get a location name
- Show a `MapCircle` overlay at the selected coordinate with the chosen radius, filled with theme color at 0.15 opacity

**Imports:** `import MapKit`, `import SwiftUI`

Follow the app's design system — use theme colors for the pin, radius circle overlay, and selected state styling. No external dependencies.

<task type="auto">
  <name>Task 2: Integrate location picker into AddHabitView</name>
  <files>Features/Habits/AddHabitView.swift</files>
  <action>
Wire the LocationPickerView into the habit creation flow.

**New state in AddHabitView:**
```
@State private var locationLatitude: Double?
@State private var locationLongitude: Double?
@State private var locationRadius: Double? = 150
@State private var locationName: String?
@State private var showLocationPicker = false
```

**New section in form — `locationSection`:**
Show this section only when `selectedVerification == .location`. Place it AFTER `verificationSection` and BEFORE `targetValueSection`.

Content:
- Section title: "LOCATION" (same `.pledgeCaption()` + `.tracking(1)` style as other sections)
- If no location set: A button "Set Location" with `location.fill` icon, styled as a tappable row in `.cleanCard()`. Tapping opens LocationPickerView as a sheet.
- If location set: Show location name, a small map preview (static `Map` at 200pt height showing the pin + radius circle), and a "Change Location" ghost button.
- Use `.cleanCard()` wrapper.

**Sheet presentation:**
```
.sheet(isPresented: $showLocationPicker) {
    LocationPickerView(
        latitude: $locationLatitude,
        longitude: $locationLongitude,
        radius: $locationRadius,
        locationName: $locationName
    )
}
```

**Update `createHabit()`:**
Pass location fields when building the Habit:
```
locationLatitude: selectedVerification == .location ? locationLatitude : nil,
locationLongitude: selectedVerification == .location ? locationLongitude : nil,
locationRadius: selectedVerification == .location ? locationRadius : nil,
locationName: selectedVerification == .location ? locationName : nil
```

**Update `resetForm()`:**
Clear location state:
```
locationLatitude = nil
locationLongitude = nil
locationRadius = 150
locationName = nil
showLocationPicker = false
```

**Update `canCreate`:**
If `selectedVerification == .location`, also require that `locationLatitude != nil` (location must be set before creating).
