- Maximum 20 monitored geofence regions per app (CLLocationManager limit)
- Geofencing requires "Always" location permission (not just "When In Use")

- Entry/exit events can be delayed by minutes (not instant)
- Background Location Updates capability needed for geofence monitoring
- UIBackgroundModes must include "location" for background delivery

<task type="auto">
  <name>Task 1: Create LocationManager service</name>
  <files>Core/Services/LocationManager.swift</files>
  <action>
Create a singleton LocationManager service following the HealthKitManager pattern:

- `@MainActor class LocationManager: NSObject, ObservableObject, CLLocationManagerDelegate`
- `static let shared = LocationManager()`
- Properties: `@Published var isAuthorized = false`, `@Published var authorizationStatus: CLAuthorizationStatus = .notDetermined`, private `locationManager: CLLocationManager`
- `requestAuthorization()` async method — requests `.authorizedAlways` permission. Must call `requestWhenInUseAuthorization()` first, then `requestAlwaysAuthorization()` (iOS requires this two-step flow).
- `startMonitoring(region: CLCircularRegion)` — adds region to monitored set. Log warning if approaching 20-region limit.
- `stopMonitoring(region: CLCircularRegion)` — removes region.
- `stopMonitoringAll()` — clears all monitored regions.
- `monitoredRegionCount: Int` computed property.
- Delegate methods:
  - `locationManager(_:didEnterRegion:)` — posts `Notification.Name("GeofenceEntry")` with region identifier in userInfo
  - `locationManager(_:didExitRegion:)` — posts `Notification.Name("GeofenceExit")` with region identifier in userInfo
  - `locationManagerDidChangeAuthorization(_:)` — updates `isAuthorized` and `authorizationStatus`
  - `locationManager(_:monitoringDidFailFor:withError:)` — print error for debugging
- Helper: `makeRegion(identifier: String, latitude: Double, longitude: Double, radius: Double) -> CLCircularRegion` — creates region with `notifyOnEntry = true`, `notifyOnExit = true`.

Use `import CoreLocation`. Do NOT use Combine — use NotificationCenter for event broadcasting (consistent with how the app communicates between services).

Private init should configure `locationManager.allowsBackgroundLocationUpdates = true` and `locationManager.pausesLocationUpdatesAutomatically = false`.
