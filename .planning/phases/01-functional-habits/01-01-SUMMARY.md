## Decisions Made
- Used `INFOPLIST_KEY_` build settings for privacy descriptions rather than an `info.properties` block in project.yml, because the project uses `GENERATE_INFOPLIST_FILE: YES` for auto-generated Info.plist. XcodeGen's `info.properties` block requires a `path` key and generates a separate plist file, which conflicts with the auto-generation approach.
- Used XcodeGen's `entitlements` section to generate `PledgeApp.entitlements` file rather than the `capabilities` section (which is not a supported XcodeGen feature).

## Deviations from Plan
- Plan specified using `capabilities` and `info.properties` blocks in project.yml. XcodeGen 2.42.0 does not support a `capabilities` block and requires a `path` in the `info` block. Used `entitlements` section for HealthKit capability and `INFOPLIST_KEY_` build settings for privacy descriptions instead. End result is functionally identical.
- Plan mentioned `PledgeApp.swift` in files_modified but no changes to PledgeApp.swift were needed for Tasks 1-2. The entitlements file `PledgeApp.entitlements` was created as an additional artifact.

## Issues Encountered
- XcodeGen 2.42.0 does not have a `capabilities` key for targets. The plan's YAML format (`capabilities: HealthKit: capabilities: [health-records]`) caused parse errors. Resolved by using the `entitlements` section instead.
- XcodeGen's `info` block requires a `path` property; omitting it causes "Decoding failed at path: Nothing found" error. Resolved by using INFOPLIST_KEY_ build settings.
