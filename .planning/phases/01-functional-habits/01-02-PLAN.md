  **Form sections using cleanCard():**

  1. **Name & Icon Section**
     - TextField for habit name (styled with glass background, white text)
     - Horizontal scrolling icon picker with common emoji options organized by category:
       - Fitness: 🏃 💪 🏋️ 🧘 🚴 🏊
       - Health: 💤 💧 🥗 🧊 💊 🫁
       - Mind: 📚 🧠 📝 🎯 ⏰ 📵

  2. **Habit Type Section**
     - Grid or list of HabitType cases with icons
     - Each type shows as a selectable pill/chip
     - Selected type highlighted with theme accent
     - Types: wakeUp, workout, steps, screenTime, sleep, meditate, noSocial, read, coldShower, journal, water, noJunkFood

     - Show only verification types relevant to selected habit type
     - Auto-select the most appropriate default based on habit type:
       - steps → healthKit
       - sleep → healthKit
       - workout → healthKit

       - wakeUp → manual
       - Others → manual
     - Options shown as selectable list items
     - Display brief description of each method (e.g., "Automatic via Apple Health", "Self-report daily")

     - Dynamic based on habit type:
       - steps: number input "Daily step goal" (default: 10000)
       - sleep: number input "Hours of sleep" (default: 7)

     - 7 day-of-week toggles (M T W T F S S)
     - Each as a circular button, filled when selected
     - Quick options: "Every day", "Weekdays", "Weekends"

  - NavigationStack with "New Pledge" title and X dismiss button
  - Use existing button styles (PrimaryCapsuleStyle, GhostButtonStyle)

<task type="auto">
  <name>Task 2: Wire AddHabitView into HabitsView and implement habit CRUD in AppState</name>
  <files>Features/Habits/HabitsView.swift, Core/AppState.swift, Core/Models.swift</files>
  <action>
  **In AppState.swift:**
  1. Add method `func addHabit(_ habit: Habit)`:
     - Append to habits array
     - Generate a TodayHabit for it if today matches the schedule
     - Persist habits
  2. Add method `func deleteHabit(_ habit: Habit)`:
     - Remove from habits array
     - Remove corresponding TodayHabit
     - Persist habits
  3. Add method `func generateTodayHabits()`:
     - Filter habits where today's weekday is in schedule
     - Create TodayHabit for each with .pending status
     - Update todayHabits array
  4. Add persistence using Codable + UserDefaults:
     - `private func saveHabits()` — encode habits array to UserDefaults key "savedHabits"
     - `private func loadHabits()` — decode from UserDefaults on init, fallback to empty array
     - Call loadHabits() in init, call saveHabits() after add/delete
     - NOTE: Remove mock data defaults — load from persistence instead. If first launch (no saved data), start with empty habits array.
  5. Update init to call loadHabits() and generateTodayHabits()

  **In Models.swift:**
  - Make Habit conform to Codable (it has simple types, should work automatically)
  - Make HabitType and VerificationType conform to Codable (they're enums with String raw values, already Codable-compatible but add explicit conformance)
  - Add Codable to HabitLog, TodayHabit, ActivityItem as well for future use

  **In HabitsView.swift:**
  1. Add `@State private var showAddHabit = false`
  2. Wire the existing + toolbar button to set `showAddHabit = true`
  3. Add `.sheet(isPresented: $showAddHabit) { AddHabitView() }`
  4. Add swipe-to-delete on habit rows: `.onDelete { appState.deleteHabit(habit) }`
     - Use a ForEach with onDelete or a swipe action on each row

  **Important:** Remove the mock habits from AppState's default initialization. Users should start with an empty habit list and add their own. Keep mock data as static vars on the model types for reference/testing but don't use them as defaults.
  </action>
  <verify>Build the project. Verify the + button presents AddHabitView. Verify creating a habit adds it to the list. Verify force-quitting and relaunching preserves habits.</verify>
  <done>+ button opens AddHabitView sheet. Creating a habit adds it to habits list and today's habits. Deleting via swipe removes it. Habits persist across app launches via UserDefaults.</done>
