<task type="auto">
  <name>Task 1: Create HabitVerificationService</name>
  <files>Core/Services/HabitVerificationService.swift</files>
  <action>
  Create `Core/Services/HabitVerificationService.swift` — a service that checks habit completion against data sources.

  **Class structure:** `@MainActor class HabitVerificationService`

  **Dependencies:** Takes HealthKitManager reference

  **Core method:** `func verifyHabit(_ habit: Habit, for date: Date) async -> VerificationResult`

  **VerificationResult** (define in this file):
  ```swift
  struct VerificationResult {
      let status: HabitStatus  // .verified or .failed
      let actualValue: Double? // what was measured
      let targetValue: Double  // what was needed
      let detail: String       // human-readable (e.g., "8,432 / 10,000 steps")
  }
  ```

  **Verification logic by type:**

  1. **steps** (verificationType: .healthKit):
     - Call healthKitManager.fetchSteps(for: date)
     - Compare against habit.targetValue
     - If steps >= target → .verified, detail: "✓ {actual} steps"
     - If steps < target → .failed, detail: "{actual} / {target} steps"

  2. **sleep** (verificationType: .healthKit):
     - Call healthKitManager.fetchSleepHours(for: date)
     - Compare against habit.targetValue (hours)
     - If hours >= target → .verified, detail: "✓ {actual}h sleep"
     - If hours < target → .failed, detail: "{actual}h / {target}h sleep"

  3. **workout** (verificationType: .healthKit):
     - Call healthKitManager.fetchWorkoutMinutes(for: date)
     - Compare against habit.targetValue (minutes)
     - If minutes >= target → .verified, detail: "✓ {actual} min workout"
     - If minutes < target → .failed, detail: "{actual} / {target} min"

  4. **wakeUp** (verificationType: .manual or .auto):
     - For now, return .pending — wake up verification needs alarm/motion detection which is complex
     - detail: "Tap to verify"

  5. **screenTime** (verificationType: .manual):
     - Cannot auto-verify without Screen Time API entitlement
     - Return .pending, detail: "Tap to verify"

  6. **All other manual types** (meditate, read, journal, coldShower, water, noJunkFood, noSocial):
     - Return .pending, detail: "Tap to verify"
     - These require user self-report

  **Error handling:** If HealthKit query fails (no permission, no data), return .pending with detail "Health data unavailable" rather than .failed. Don't penalize users for permission issues.

  **Batch method:** `func verifyAllHabits(_ habits: [Habit], for date: Date) async -> [UUID: VerificationResult]`
  - Run all verifications concurrently using TaskGroup
  - Return dictionary keyed by habit ID
  </action>

<task type="auto">
  <name>Task 2: Wire verification into AppState and HomeView</name>
  <files>Core/AppState.swift, Features/Home/HomeView.swift, PledgeApp.swift</files>
  <action>
  **In AppState.swift:**
  1. Add property: `let verificationService = HabitVerificationService(healthKitManager: HealthKitManager.shared)`
  2. Add method `func requestHealthPermissions() async`:
     - Call HealthKitManager.shared.requestAuthorization()
     - Only request if any habits use .healthKit verification
  3. Add method `func verifyTodayHabits() async`:
     - Call verificationService.verifyAllHabits() for today's habits that are still .pending
     - Update todayHabits array with results (status + detail)
     - For .failed habits: add stakeAmount to investmentPoolValue, create ActivityItem
     - For .verified habits: update streak, create ActivityItem
     - Save state after verification
  4. Add method `func manuallyVerifyHabit(_ todayHabitId: UUID)`:
     - Find the TodayHabit, set status to .verified
     - Set verifiedAt to Date()
     - Update detail to "Verified {time}"
     - Update streak count
     - Save state
  5. Add property `@Published var isVerifying = false` for loading state

  **In HomeView.swift:**
  1. On .onAppear or .task, call `appState.verifyTodayHabits()` (with isVerifying loading state)
  2. Show a subtle loading indicator on habits being auto-verified
  3. Wire "Verify Now" button to `appState.manuallyVerifyHabit(todayHabit.id)` for manual types
  4. For HealthKit-type habits that are still pending, show a "Refresh" button that re-runs verification
  5. Update habit row to show the verification detail string from VerificationResult

  **In PledgeApp.swift:**
  1. On app launch (in .task or .onAppear after auth), request health permissions if needed
  2. Call `appState.requestHealthPermissions()` then `appState.verifyTodayHabits()`
  3. Add `.onReceive(NotificationCenter.default.publisher(for: UIApplication.willEnterForegroundNotification))` to re-verify when app returns from background

  **Stake/Investment flow on failure:**
  - When a habit fails, its stakeAmount is added to investmentPoolValue
  - Create an ActivityItem: icon "📈", title "Stake invested", detail "$X from {habitName}", isFailure: true
  - This is the "Miss your habit, fund your future" mechanic
