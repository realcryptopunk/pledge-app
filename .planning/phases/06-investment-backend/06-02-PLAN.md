  <files>Core/Services/InvestRelayerService.swift, Core/AppState.swift</files>
  <action>
    **1. Create `Core/Services/InvestRelayerService.swift`:**

    A simple service that calls the invest-relayer edge function. Follow the pattern from CoinbaseOnrampView's OnrampService.

    ```swift
    struct InvestRelayerResponse {
        let txHash: String
        let explorerURL: String
    }
    ```

    Static function `callInvestRelayer(userWallet: String, usdcAmount: Double) async throws -> InvestRelayerResponse`:
    - Build URL: `"\(EnvConfig.supabaseURL)/functions/v1/invest-relayer"`
    - POST with JSON body: `{ "user_wallet": userWallet, "usdc_amount": usdcAmount }`
    - Parse response JSON for `tx_hash` and `explorer_url`
    - On non-200 status: throw descriptive error with response body
    - On success: return InvestRelayerResponse

    **2. Modify `Core/AppState.swift`:**

    Find the habit failure handling code (where `result.status == .failed` and the stake is allocated locally). After the existing local allocation logic, add an async call to the relayer:

    ```swift
    // After local allocation (existing code)...

    // Fire on-chain investment via relayer
    if let walletAddress = privyManager.walletAddress {
        Task {
            do {
                let result = try await InvestRelayerService.callInvestRelayer(
                    userWallet: walletAddress,
                    usdcAmount: stakeAmount
                )
                // Update toast to include tx hash
                await MainActor.run {
                    investmentToast = "$\(Int(stakeAmount)) invested on-chain — view on explorer"
                    // Optionally store the explorer URL for tapping
                }
                print("[InvestRelayer] Success: \(result.txHash)")
            } catch {
                // Non-blocking — log the error but don't fail the habit flow
                print("[InvestRelayer] Failed: \(error.localizedDescription)")
            }
        }
    }
    ```

    Key points:
    - The relayer call is FIRE-AND-FORGET in a detached Task — habit failure UI flow should not block on the on-chain tx
    - If the relayer call fails (no approval, insufficient balance, network issues), the local allocation still works — on-chain is best-effort for the demo
    - Log success/failure for debugging
    - Update the investmentToast message to mention "on-chain" when the tx succeeds
    - Store `result.explorerURL` if there's a place to show it (e.g., in the activity feed or as a tappable link)

    Avoid: Do NOT make the on-chain call blocking — the user shouldn't wait for tx confirmation to see the habit failure result. Do NOT remove the existing local allocation logic — keep it as the primary UX, on-chain is additive. Do NOT add new dependencies.
