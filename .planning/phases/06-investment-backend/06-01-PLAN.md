**Contract ABI (relevant function):**
```solidity
function investForUser(address user, uint256 usdcAmount) external onlyRelayer nonReentrant
```
- Pulls USDC from `user` (requires prior ERC-20 approval to vault)
- Takes 2% platform fee → treasury
- Distributes 98% across stock allocations (TSLA/AMZN/PLTR/AMD)
- Emits `Invested(user, usdcAmount, fee)`

**Existing edge function pattern (from coinbase-session-token):**
- Deno serve() handler with CORS

- Env vars via Deno.env.get()
- Structured error responses with status codes

  <name>Task 1: Create invest-relayer Supabase edge function</name>
  <files>supabase/functions/invest-relayer/index.ts</files>
  <action>
    Create `supabase/functions/invest-relayer/index.ts` following the existing edge function pattern from coinbase-session-token.

    The function:
    1. POST handler with CORS headers (same pattern as coinbase-session-token)
    2. Accepts JSON body: `{ user_wallet: string, usdc_amount: number }` where usdc_amount is in human-readable units (e.g., 5.00 for $5 USDC)
    3. Input validation:
       - user_wallet: valid Ethereum address (0x + 40 hex chars)
       - usdc_amount: positive number, max 1000 (safety cap for testnet)
    4. Load env vars:
       - `RELAYER_PRIVATE_KEY` — private key of the relayer wallet (same wallet that deployed the contract)
       - `RH_TESTNET_RPC_URL` — defaults to `https://rpc.testnet.chain.robinhood.com` if not set
    5. Use `npm:ethers@6` (Deno npm specifier) to:
       a. Create JsonRpcProvider with RH testnet RPC
       b. Create Wallet from RELAYER_PRIVATE_KEY, connected to provider
       c. Create Contract instance for PledgeVaultRH at `0x70d73b04d0c2ee4f73eb44cbf5377a2c3fbc52ba`
       d. Use minimal ABI: `["function investForUser(address user, uint256 usdcAmount) external"]`
       e. Query USDC decimals: create USDC contract at `0xbf4479C07Dc6fdc6dAa764A0ccA06969e894275F` with `["function decimals() view returns (uint8)"]`, call decimals(), use result to parse amount. If decimals() call fails, default to 18.
       f. Convert usdc_amount to on-chain units: `ethers.parseUnits(usdc_amount.toString(), decimals)`
       g. Call `vault.investForUser(user_wallet, parsedAmount)` — this sends a transaction
       h. Wait for tx receipt (1 confirmation)
    6. Return JSON response:
       ```json
       {
         "success": true,
         "tx_hash": "0x...",
         "explorer_url": "https://explorer.testnet.chain.robinhood.com/tx/0x..."
       }
       ```
    7. Error handling:
       - Missing env vars → 500 with clear message
       - Invalid input → 400
       - Transaction revert → 502 with revert reason (e.g., "PledgeVault: not relayer", "ERC20: insufficient allowance")
       - RPC errors → 502 with details
       - Wrap all in try/catch with structured error response

    Security:
    - Do NOT log the private key
    - Do NOT include the private key in error responses
    - The function is "public" (no JWT verification, matches existing edge function pattern) — the contract's onlyRelayer modifier provides access control
    - Amount cap at 1000 USDC as testnet safety guardrail

    Avoid: Do NOT use ethers v5 syntax (v6 has breaking changes: parseUnits is top-level, Contract constructor changed). Do NOT hardcode USDC decimals — query from contract with fallback. Do NOT use `import { createClient } from '@supabase/supabase-js'` — this function doesn't need Supabase client, just ethers for on-chain interaction.
